## Implementation Order (Revised)

### Step 1: Env ‚Äî Add CEB_API_BASE
**Files:** `extension/.env`, `extension/.example.env`, `extension/packages/env/lib/types.ts`
- Add `CEB_API_BASE=http://localhost:3000` to both `.env` and `.example.env`
- Add `readonly CEB_API_BASE: string` to `ICebEnv` interface in `packages/env/lib/types.ts`

### Step 2: Shared Types ‚Äî Add MARK types
**File:** `extension/packages/shared/lib/utils/types.ts`
- Add interfaces: `RepoInfo`, `TerminalLine`, `ScanState`, `BuildIdea`
- Export them from the existing `types.ts`

### Step 3: Shared Constants ‚Äî Replace with MARK constants
**File:** `extension/packages/shared/const.ts`
- Replace boilerplate content with MARK constants
- Read `CEB_API_BASE` from env via `@extension/env`
- Export: `API_BASE`, `FORGE_ENDPOINT`, `OPPORTUNITIES_ENDPOINT`, `MAX_SCANS_PER_DAY`

### Step 4: Manifest ‚Äî Configure for MARK
**File:** `extension/chrome-extension/manifest.ts`
- Change `name` to `'MARK Intelligence'`
- Change `description` to `'Scan any GitHub repo. Get a full intelligence report + 3 pump.fun-ready build ideas.'`
- Add `'downloads'` to `permissions` array
- Keep `<all_urls>` host permissions
- Remove unused: `options_page`, `chrome_url_overrides.newtab`, `devtools_page`, `side_panel`, example.com content scripts

### Step 5: Content Script ‚Äî GitHub URL detection
**File:** `extension/pages/content/src/matches/all/index.ts`
- On mount: extract `owner`/`repo` from `location.pathname`, save to `chrome.storage.local`
- Skip non-repo paths: `/explore`, `/topics`, `/trending`, `/marketplace`, `/settings`, `/orgs`
- If not a valid repo, remove `currentRepo` from storage
- `MutationObserver` on `document.body` to catch GitHub SPA navigation (PJAX)
- Compare `lastPath` vs current to avoid duplicate writes

### Step 6: Service Worker ‚Äî SSE streaming
**File:** `extension/chrome-extension/src/background/index.ts`
- Replace entire file with MARK service worker
- Listen for messages: `START_SCAN`, `RESET_STATE`
- `startScan()`: POST to FORGE_ENDPOINT, read SSE via ReadableStream, update storage per event
- `handleSSEEvent()`: switch on `data.type` ‚Üí update `scanState` fields
  - `status` ‚Üí append `TerminalLine` (cls: blue)
  - `stack` ‚Üí set `tags`
  - `skills` ‚Üí set `skills` array
  - `content` ‚Üí append to `markFile`
  - `done` ‚Üí set `status: 'done'`, `zipBase64`, `markFile`, append done terminal line
  - `error` ‚Üí set `status: 'error'`, `error` message
- After scan done ‚Üí non-blocking POST to OPPORTUNITIES_ENDPOINT ‚Üí update `ideas`
- Handle 429 ‚Üí `status: 'rate_limited'`, broadcast
- `getState()` / `updateState()` helpers for chrome.storage access
- `broadcast()` via `chrome.runtime.sendMessage` (silent catch for closed popups)

### Step 7: Popup CSS ‚Äî Design tokens + component styles
**File:** `extension/pages/popup/src/Popup.css`
- Replace with MARK design tokens as CSS custom properties
- Styles for: `.popup-root`, `.popup-header`, `.glass-zone`, `.glass-canvas`, `.glass-overlay`, `.scan-status`, `.chip-row`, `.chip`, `.idle-state`, `.scan-btn`, `.scan-progress`, `.tabs`, `.tab`, `.tab-content`, `.terminal`, `.term-line`, `.term-prefix`, `.cursor`, `.file-list`, `.file-row`, `.idea-card`, `.popup-footer`, `.no-repo`, `.rate-limit-msg`, `.idea-name`, `.idea-ticker`, `.idea-desc`, `.idea-pf`, `.idea-effort`, `.ideas-loading`

### Step 8: Popup global CSS ‚Äî Update dimensions
**File:** `extension/pages/popup/src/index.css`
- Change `width` from `300px` to `360px`
- Change `height` from `260px` to `min-height: 480px`
- Add MARK font imports: Geist via Google Fonts `@import`

### Step 9: Tailwind config ‚Äî Extend with MARK colors
**File:** `extension/pages/popup/tailwind.config.ts`
- Extend theme colors with MARK palette: ink, paper, surface, border, muted, accent, accent-bg, accent-border, green, green-bg, green-border, term-bg, term-text, term-green, term-blue, term-yellow, term-dim

### Step 10: Popup UI ‚Äî Full implementation
**File:** `extension/pages/popup/src/Popup.tsx`
- Replace entire file with MARK popup
- States handled:
  - **No repo:** show "Open any GitHub repository to use MARK"
  - **Idle + repo:** show repo name, "Scan this repo" button, remaining count
  - **Scanning:** glass zone animation, terminal streaming, stack chips, skill badge
  - **Done (Mark File tab):** terminal output, CLAUDE.md + ZIP download rows
  - **Done (Build Opportunities tab):** 3 idea cards with stagger animation
  - **Rate limited:** "5/5 scans today" message
  - **Error:** error message with retry option
- Components (in-file):
  - `<Popup />` ‚Äî orchestrator, storage reads, message listening, state management
  - `<GlassZone />` ‚Äî canvas + glass overlay with status + chips
  - `<GlassCanvas />` ‚Äî canvas 2D liquid glass component (no mouse tracking)
  - `<IdeaCard />` ‚Äî single idea card with stagger entrance
- Storage reads: `chrome.storage.local.get(['currentRepo', 'scanState'])`
- Message listener: `chrome.runtime.onMessage.addListener` for broadcasts
- Download: `chrome.downloads.download({ url: blobURL, filename })`
- Rescan: send `RESET_STATE` then `START_SCAN` messages

## Potential Issues

| Issue | Mitigation |
|-------|------------|
| Service worker killed during scan | All state persisted to chrome.storage.local after each event |
| Popup closed mid-scan | State persists in storage; next popup open reads current state |
| GitHub SPA navigation (PJAX) | MutationObserver catches pathname changes; compare `lastPath` |
| CORS from chrome-extension:// | API `/api/forge` must allow `chrome-extension://*` origin |
| `chrome.downloads` API requires user gesture | Download triggered from button click handler |
| SSE events arrive fast (many content chunks) | Single `updateState()` per event; popup reads fresh state from storage |
| Extension CSP blocks inline styles | Use style objects or CSS classes, never inline style tags/attributes in HTML |
| Liquid glass canvas performance | `requestAnimationFrame`, limit blob count to 4-5, reduce resolution if needed |
| Boilerplate Vite config port conflict | Use different port if `:3000` is already taken by Next.js dev server |
| Boilerplate has `Turbo` monorepo build | `pnpm dev` at root runs all packages; popup page config at `pages/popup/vite.config.mts` |
| `@extension/env` uses `process.env.CEB_*` | Env vars must be prefixed with `CEB_`; accessed via `import env from '@extension/env'` |
| `packages/shared` is ESM (.mts + .js) | Type exports need `export type *` pattern; import paths use `.js` extension |
